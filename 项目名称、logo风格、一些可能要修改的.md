# 项目名称、Logo风格与执行清单（离线纯前端版，v1.5）

## 1. 项目名称

项目名：读者之旅 Reader’s Journey

命名含义：
1. 读者：目标人群清晰。
2. 之旅：强调成长路径和阶段性里程碑。
3. Reader’s Journey：便于后续多语言延展。

Slogan：
1. 中文：把阅读，变成一场冒险。
2. 英文：Turn reading into an adventure.

## 2. 视觉风格建议

1. Logo：一本翻开的书，书页延伸为路径或星轨，表达“阅读旅程”。
2. 色彩：深海蓝 + 铜金高光 + 羊皮浅米辅助层。
3. 质感：轻纹理背景 + 卡片层级 + 卷轴书架意象，避免网页模板感。
4. 交互风格：中强反馈、快响应，动效短而明确。

## 3. 数值策略（可持续迭代）

1. 历史书单：前10本100%，10-50本70%，50+本30%。
2. 新增书奖励：x1.5。
3. 每日录入节奏：第1本100%，第2本80%，第3本及以上50%。
4. 自编录入：x0.7（防止滥用）。

## 4. 当前技术路线（已确认）

1. 纯前端离线项目。
2. 不使用后端服务，不使用 Supabase，不做登录。
3. 不做推荐购书与商业化导流。
4. 不做 AI 书童助手。
5. 页面范围固定为：录入、面板、分享、设置（进度能力并入面板）。

## 5. 立即执行清单（离线 MVP）

1. 维护离线书库与本地搜索索引。
2. 保持 5 万本真实书籍规模（中文热门优先），来源可追溯。
3. 录入闭环：搜索选择入库 + 自编录入兜底 + 分类必选。
4. 面板闭环：卷轴书架 + 进度更新奖励 + 感触时间线。
5. 设置闭环：隐私入口、导出/导入/重置、版本与本地存储状态提示。
6. 统一一屏布局规则：摘要展示 + 底部弹层查看详情。
7. 完成断网验证：无网状态下核心流程可用。

## 6. 素材资源策略（音乐/图像/Logo）

### 6.1 原则
1. 必须可商用且可追溯授权。
2. 首版轻量优先，非必要素材不引入。
3. 统一风格，减少混搭。

### 6.2 图像与图标
1. 优先使用开源图标库或项目内自制 SVG。
2. Logo 可用 AI 生成初稿，再人工统一风格并输出 SVG。
3. 书籍封面非必需，可不展示。

### 6.3 音频
1. 采用长音环境氛围 BGM（双轨循环）。
2. 保留轻提示音（录入成功/解锁/升级）。
3. 音频默认可开关并支持音量调节。

### 6.4 动效
1. 优先 CSS/JS 轻动效（结算反馈、进度变化、卡片高亮）。
2. 复杂动画通过 Sheet 或局部效果表达，避免主流程卡顿。
3. 单次反馈动效控制在 1 秒级。

## 7. 资源合规清单（必须维护）

每个外部素材都要记录：
1. 来源链接
2. 作者/平台
3. 许可证类型
4. 是否可商用
5. 是否需要署名
6. 项目内使用位置

## 8. 后续可讨论项（非当前必做）

1. 是否升级到 IndexedDB（增强大数据量表现）。
2. 是否补充离线书库扩展包导入机制。
3. 是否增加中英文双语文案切换。
